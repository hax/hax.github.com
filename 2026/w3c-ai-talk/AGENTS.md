# AGENTS.md

本目录为 slides 讲稿。

slide 采用高桥流风格。

高桥流内容在文本块中，不在块中的是对上一个块的备注。

## 高桥流

- 保持短句、少字、大字感，优先突出一个核心信息。
- 单页只表达一个重点；如果出现并列信息，优先拆成多页。
- 备注区用于补充背景、例子、过渡和口播，要对应 slide 正文。
- 新增 slide 时，先写代码块，再在其后补充必要备注。
- 正文常用 1 到 3 行短句，优先用名词、判断、对比、年份、标签，而不是完整长句。
- 措辞要像口播里的“重音词”，可以故意压缩、省略、留白，让演讲者在备注里展开。
- 常见写法是先给一个大词或问题，再逐页补一个例子、一个数字、一个反转、一个结论。
- 页与页之间要有明显的“推进感”：定义 -> 例子 -> 预期 -> 现实 -> 新结论，或 身份 -> 资历 -> 经历 -> 转折 -> 现在。
- 相邻页正文要尽量形成承接、递进或反差，避免只是同类信息平铺罗列。
- 关键转折页要特别短，像“现实……”“先驱 / 变先烈”这种，用来停顿、换气、翻篇。
- 备注不是重复 slide 正文，而是解释上一页为什么成立、补足限定条件、交代具体事实，或把话题自然引到下一页。
- 如果正文用了高度压缩的判断，备注里要负责“解压”，把时间、对象、范围、证据说清楚。
- 如果正文是时间点、标题式短语或并列词，备注里通常补一两句口播，让观众知道这页为什么重要。
- 不要机械追求“每页都极简”。为了放原文、证据、对照或表格，必要时可以有一两页更复杂的 slide，但仍要服务当前段落的叙事与论证。
- 新增 slides 时，先想清楚这一页在整段叙事里的作用，再决定它是“立题页”“展开页”“转折页”还是“落点页”。
